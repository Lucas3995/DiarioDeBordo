# Plan 4: Esboço dos BCs de Suporte e Contratos de Integração

## Goal
Criar 6 documentos de esboço para os BCs de suporte (Reprodução, Integração Externa, Busca, Portabilidade, Identidade, Preferências), documentando responsabilidades, contratos de interface e eventos de domínio — sem modelagem tática completa (adiada para as fases respectivas).

## Context
A decisão D-03 (CONTEXT.md) determina que BCs de suporte recebem esboço nesta fase — modelagem tática completa é adiada para quando cada BC entrar em produção. O esboço formaliza os contratos de interface já identificados na pesquisa (01-RESEARCH.md seção 1.4) para que o walking skeleton da Phase 2 possa ser implementado corretamente. Depende dos Planos 02 e 03 para ter as interfaces já documentadas no lado do Acervo e Agregação como referência.

**`docs/domain/reproducao-esboco.md`:**

- Classificação: Suporte
- Responsabilidade: Consumir conteúdo internamente (texto puro/Markdown/HTML, áudio local, vídeo embed YouTube) e oferecer marcação de progresso. Não armazena dados próprios — lê Conteudo via interface readonly.
- O que NÃO faz: não armazena cópias de conteúdo; não faz download de vídeos; não substitui players dedicados (VLC, Spotify)
- Interfaces consumidas:
  ```csharp
  // Implementada em Module.Acervo:
  public interface IConteudoParaReproducaoProvider
  {
      Task<ConteudoReproducaoDto?> ObterAsync(Guid id, Guid usuarioId, CancellationToken ct);
  }
  public sealed record ConteudoReproducaoDto(
      Guid Id, string Titulo, FormatoMidia Formato,
      IReadOnlyList<FonteOrdenada> FontesOrdenadas, IReadOnlyList<GanchoDto> Ganchos);
  ```
- Interfaces publicadas: nenhuma — não implementa interfaces para outros BCs
- Eventos publicados:
  - `ReproducaoIniciadaNotification(ConteudoId, Posicao)` — quando reprodução começa → ViewModels podem oferecer marcação de progresso
  - `ReproducaoConcluidaNotification(ConteudoId)` — quando conteúdo termina → oferecer marcação de conclusão
- Esses eventos disparam `PersistirProgressoCommand` que vai ao BC Acervo via MediatR — Module.Reproducao não escreve diretamente no banco
- Adiado: implementação concreta dos renderers (Markdown, HTML, embed YouTube), player de áudio, lógica de ganchos automáticos

**`docs/domain/integracao-externa-esboco.md`:**

- Classificação: Suporte
- Responsabilidade: Implementar adaptadores de plataformas externas (RSS, YouTube) que buscam itens de feed e metadados de conteúdo. Cada adaptador implementa `IAdaptadorPlataforma`.
- O que NÃO faz: não persiste dados; não decide o que mostrar ao usuário; não aplica ranqueamento
- Interfaces publicadas (implementadas aqui, consumidas por Module.Agregacao):
  ```csharp
  public interface IAdaptadorPlataforma
  {
      bool SuportaPlataforma(string tipo);
      Task<Result<IReadOnlyList<ItemFeedDto>>> MontarFeedAsync(
          FonteSubscricao fonte, PaginacaoParams paginacao, CancellationToken ct);
      Task<Result<MetadadosExternosDto?>> ObterMetadadosAsync(
          string urlOuIdentificador, string plataforma, CancellationToken ct);
  }
  public sealed record MetadadosExternosDto(
      string? Titulo, string? Descricao, string? ThumbnailUrl, string? Autor);
  ```
- Restrições de segurança invioláveis (Padrões Técnicos v4, seção 4.2):
  - `DtdProcessing.Prohibit` em todo XML/RSS — sem exceção
  - Limite 5MB por payload, timeout 10s
  - Rejeitar URLs que resolvam para IPs privados (SSRF prevention)
  - Whitelist de protocolos: `https`, `http`, `file` apenas
- Adiado: implementação dos adaptadores RSS e YouTube, metadados de plataforma, tratamento de throttling/rate limits

**`docs/domain/busca-esboco.md`:**

- Classificação: Suporte
- Responsabilidade: Fornecer busca textual e filtros combinados sobre os conteúdos do usuário, incluindo operações em lote.
- O que NÃO faz: não indexa conteúdo externo; não recomenda conteúdo; não faz busca semântica
- Interface publicada (implementada aqui):
  ```csharp
  public interface IBuscaConteudoService
  {
      Task<Result<PaginatedList<ConteudoResumoDto>>> BuscarAsync(
          Guid usuarioId, FiltroBusca filtro, PaginacaoParams paginacao, CancellationToken ct);
      Task<Result<int>> ExecutarOperacaoEmLoteAsync(
          Guid usuarioId, IReadOnlyList<Guid> ids, OperacaoLote operacao, CancellationToken ct);
  }
  public sealed record FiltroBusca(
      string? TextoLivre,
      FormatoMidia? Formato,
      PapelConteudo? Papel,
      TipoColetanea? TipoColetanea,
      Guid? CategoriaId,
      decimal? NotaMinima,
      EstadoProgresso? Progresso,
      DateOnly? DataCriacaoApos,
      Guid? FonteId
  );
  ```
- Adiado: implementação de full-text search (PostgreSQL tsvector vs. extensão), índices, operações em lote concretas

**`docs/domain/portabilidade-esboco.md`:**

- Classificação: Suporte
- Responsabilidade: Orquestrar exportação e importação de dados do usuário entre instâncias. Não tem dados próprios — agrega dados de outros módulos via `IDadosExportaveisProvider`.
- O que NÃO faz: não armazena dados; não sincroniza em tempo real; não faz backup automático
- Interfaces consumidas (cada módulo com dados exportáveis implementa):
  ```csharp
  // Implementada em Module.Acervo e Module.Preferencias:
  public interface IDadosExportaveisProvider
  {
      Task<ExportacaoDto> ExportarAsync(Guid usuarioId, CancellationToken ct);
  }
  ```
- Responsabilidade do BC Portabilidade: consumir IDadosExportaveisProvider de todos os módulos, serializar para formato agnóstico de plataforma (legível por humano), e importar de volta
- Adiado: formato concreto de exportação (JSON estruturado vs. CSV vs. outro), estratégia de resolução de conflitos na importação, checksum de integridade

**`docs/domain/identidade-esboco.md`:**

- Classificação: Genérico
- Responsabilidade: Autenticação local multi-usuário, roles (consumidor/admin), e fornecer identidade do usuário atual para todos os módulos.
- O que NÃO faz: sem OAuth/SSO; sem autenticação remota; sem perfis públicos
- Interface publicada (consumida por TODOS os módulos):
  ```csharp
  // Definida em Core — consumida por todos os módulos que precisam do usuário atual:
  public interface IUsuarioAutenticadoProvider
  {
      Guid? UsuarioIdAtual { get; }
      IReadOnlySet<Role> RolesAtuais { get; }
  }
  public enum Role { Consumidor, Admin }
  ```
- Invariante de segurança CRÍTICA: Área admin não existe para usuários sem role `Admin`. Qualquer tentativa de acesso retorna comportamento genérico — sem indicação de que a área existe. Esta regra é verificada na camada de serviço (não apenas UI).
- Hash de senhas: Argon2id (conforme ADR-005)
- Adiado: modelo concreto de User, grupos de roles, tela de administração, fluxo de criação de usuários

**`docs/domain/preferencias-esboco.md`:**

- Classificação: Genérico
- Responsabilidade: Armazenar e fornecer preferências de interface por usuário (tema, fonte, acessibilidade, paginação, disclosure progressivo, configurações de uso saudável).
- O que NÃO faz: não aplica as preferências — apenas as armazena e fornece
- Interface publicada (consumida por UI e por módulos que precisam respeitar preferências):
  ```csharp
  public interface IPreferenciasProvider
  {
      Task<Tema> ObterTemaAsync(Guid usuarioId, CancellationToken ct);
      Task<int> ObterItensPorPaginaAsync(Guid usuarioId, CancellationToken ct);
      Task<bool> DisclosureProgressivoAtivoAsync(Guid usuarioId, CancellationToken ct);
      Task<ConfiguracaoUsoSaudavel> ObterUsoSaudavelAsync(Guid usuarioId, CancellationToken ct);
  }
  public sealed record ConfiguracaoUsoSaudavel(
      bool MonitoramentoTempoAtivo,
      TimeSpan? LimiteDiario,
      bool EscalaCinzaAtiva,
      bool LembretesAtivos
  );
  ```
- Implementa `IDadosExportaveisProvider` (preferências do usuário são exportáveis)
- Adiado: preferências de admin (defaults globais), modelo concreto de preferências de fonte (família, tamanho, cor), temas customizáveis
