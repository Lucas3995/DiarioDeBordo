## Task 4.1: Create dotnet-tools.json and CI workflow

**Description:**  
Create the dotnet tools manifest for `reportgenerator` and the GitHub Actions CI workflow. The workflow mirrors the validation commands from `02-RESEARCH.md` section 8.2. The coverage gate uses `xmllint` on Linux and PowerShell `Select-Xml` on Windows to parse the Cobertura XML.

**Files to create:**

### `.config/dotnet-tools.json`
```json
{
  "version": 1,
  "isRoot": true,
  "tools": {
    "dotnet-reportgenerator-globaltool": {
      "version": "5.4.4",
      "commands": [
        "reportgenerator"
      ]
    }
  }
}
```

### `.github/workflows/ci.yml`
```yaml
name: CI

on:
  push:
    branches: [ main, teste-com-gsd ]
  pull_request:
    branches: [ main ]

env:
  DOTNET_SKIP_FIRST_TIME_EXPERIENCE: 1
  DOTNET_CLI_TELEMETRY_OPTOUT: 1
  DOTNET_NOLOGO: true

jobs:
  build-and-test:
    name: Build & Test (${{ matrix.os }})
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false   # See both platforms fail, not just the first
      matrix:
        os: [ ubuntu-latest, windows-latest ]

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup .NET 9
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: '9.0.x'

      - name: Restore tools (reportgenerator)
        run: dotnet tool restore

      - name: Restore dependencies
        run: dotnet restore DiarioDeBordo.sln

      # TreatWarningsAsErrors=true in Directory.Build.props — warnings fail the build
      - name: Build (Release, zero warnings)
        run: dotnet build DiarioDeBordo.sln --no-restore --configuration Release

      # Unit and domain tests run without infrastructure
      - name: Run unit tests
        run: >
          dotnet test tests/Tests.Unit/Tests.Unit.csproj
          --no-build
          --configuration Release
          --collect:"XPlat Code Coverage"
          --results-directory ./coverage/unit
          --logger "trx;LogFileName=unit-results.trx"

      - name: Run domain invariant tests
        run: >
          dotnet test tests/Tests.Domain/Tests.Domain.csproj
          --no-build
          --configuration Release
          --collect:"XPlat Code Coverage"
          --results-directory ./coverage/domain
          --logger "trx;LogFileName=domain-results.trx"

      # Integration tests use Testcontainers — Docker must be available
      # ubuntu-latest has Docker; windows-latest requires specific runner config
      - name: Run integration tests (Linux)
        if: runner.os == 'Linux'
        run: >
          dotnet test tests/Tests.Integration/Tests.Integration.csproj
          --no-build
          --configuration Release
          --collect:"XPlat Code Coverage"
          --results-directory ./coverage/integration
          --logger "trx;LogFileName=integration-results.trx"

      - name: Run E2E tests (Linux)
        if: runner.os == 'Linux'
        run: >
          dotnet test tests/Tests.E2E/Tests.E2E.csproj
          --no-build
          --configuration Release
          --collect:"XPlat Code Coverage"
          --results-directory ./coverage/e2e
          --logger "trx;LogFileName=e2e-results.trx"

      # Merge coverage reports from all test runs
      - name: Generate coverage report
        run: >
          dotnet tool run reportgenerator
          -reports:"./coverage/**/coverage.cobertura.xml"
          -targetdir:"./coverage/report"
          -reporttypes:"HtmlInline;Cobertura;Badges"

      # Coverage gate: fail if line coverage < 95%
      - name: Check coverage threshold (Linux)
        if: runner.os == 'Linux'
        run: |
          COVERAGE=$(xmllint --xpath "string(//coverage/@line-rate)" ./coverage/report/Cobertura.xml)
          echo "Line coverage: $COVERAGE"
          if (( $(echo "$COVERAGE < 0.95" | bc -l) )); then
            echo "::error::Coverage $COVERAGE is below the 95% threshold."
            exit 1
          fi
          echo "::notice::Coverage $COVERAGE meets the ≥95% threshold."

      - name: Check coverage threshold (Windows)
        if: runner.os == 'Windows'
        shell: pwsh
        run: |
          [xml]$xml = Get-Content ./coverage/report/Cobertura.xml
          $lineRate = [double]$xml.coverage.'line-rate'
          Write-Host "Line coverage: $lineRate"
          if ($lineRate -lt 0.95) {
            Write-Error "Coverage $lineRate is below the 95% threshold."
            exit 1
          }
          Write-Host "Coverage $lineRate meets the >=95% threshold."

      # Upload coverage for review (does not fail the build)
      - name: Upload coverage report
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: coverage-report-${{ matrix.os }}
          path: ./coverage/report/
          retention-days: 14

      # Upload test results for review
      - name: Upload test results
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: test-results-${{ matrix.os }}
          path: ./coverage/**/*.trx
          retention-days: 7

      # Publish to Codecov (optional, needs CODECOV_TOKEN secret)
      - name: Upload to Codecov
        uses: codecov/codecov-action@v4
        if: runner.os == 'Linux'
        with:
          files: ./coverage/report/Cobertura.xml
          fail_ci_if_error: false   # Don't break CI if Codecov is down
        env:
          CODECOV_TOKEN: ${{ secrets.CODECOV_TOKEN }}
```

**Notes on the workflow design:**
- Integration and E2E tests run on Linux only because they use Testcontainers which requires Docker. GitHub-hosted Windows runners support Docker in containers but configuration is complex. Windows tests cover unit + domain which have no infrastructure dependencies.
- `fail-fast: false` ensures both platforms report — don't let a Linux failure hide a Windows failure.
- `DOTNET_CLI_TELEMETRY_OPTOUT` and `DOTNET_NOLOGO` reduce noise in CI logs.
- The coverage gate uses `xmllint` on Linux (installed by default on ubuntu) and PowerShell XML parsing on Windows — both parse the Cobertura XML from `reportgenerator`.

**Acceptance Criteria:**
- `.github/workflows/ci.yml` is valid YAML (`python -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))"`)
- Workflow triggers defined for both push and PR
- Matrix has exactly `ubuntu-latest` and `windows-latest`
- Coverage gate step is present for both OS variants
- `.config/dotnet-tools.json` specifies `dotnet-reportgenerator-globaltool`
