# Phase 4 — UI Design Contract

> Visual and interaction contract for Curadoria -- Coletaneas e Fontes. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** Coletaneas (Guiada e Miscelanea) com composicao e protecao contra ciclos, anotacoes contextuais, fontes com prioridade/fallback, imagens de capa, deduplicacao com aviso na criacao.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Avalonia UI -- not web) |
| Preset | not applicable |
| Component library | Avalonia FluentTheme |
| Icon library | Avalonia built-in icons (FluentIcons via PathIcon) + unicode emoji for semantic indicators |
| Font | System default (Segoe UI on Windows, system sans-serif on Linux -- FluentTheme default) |

**Continuity:** This contract extends Phase 3 UI-SPEC. All tokens, patterns, and conventions from Phase 3 carry forward unless explicitly overridden below.

## Spacing Scale

Inherited from Phase 3. No changes to the base scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, chip internal padding vertical |
| sm | 8px | Between buttons, compact element spacing, card vertical gap |
| md | 12px | Form field group spacing within inline forms |
| lg | 16px | Card internal padding, section content padding |
| xl | 24px | Page-level horizontal margin, section separator spacing |
| 2xl | 32px | Modal internal padding (horizontal) |
| 3xl | 48px | Reserved for page-level spacing |

Exceptions:
- Touch target minimum: 44x44px for all interactive elements (WCAG 2.5.8 AAA)
- Arrow reorder buttons (sources): 32x32px visual with 44x44px effective hit area via padding
- Breadcrumb separator spacing: 4px horizontal between each segment
- Thumbnail on collection card: 48x48px (fixed square, object-fit cover)
- Cover image preview in modal: 120x160px (3:4 aspect ratio, portrait orientation for book-like content)

## Scientific Foundation (D-18)

### Research Basis

The interaction patterns for this phase are grounded in peer-reviewed literature and established HCI research:

1. **Shneiderman, B. (1983).** "Direct manipulation: A step beyond programming languages." *IEEE Computer, 16*(8), 57-69. -> Reordering sources via arrow buttons (up/down) follows the direct manipulation principle: continuous representation of objects, rapid reversible actions, physical gestures replacing typed commands. Arrow buttons chosen over drag-and-drop for keyboard accessibility (WCAG 2.1.1).

2. **Shneiderman, B. & Plaisant, C. (2010).** *Designing the User Interface.* 5th ed. -> Progressive disclosure for contextual annotations: the annotation field is hidden until the user explicitly requests it (icon button per item). Reduces visual clutter in the item list within the collection modal.

3. **Myers, B. A. (1990).** "Taxonomies of visual programming and program visualization." *Journal of Visual Languages and Computing, 1*(1), 97-123. -> Sequential progress indicators (numbered items in Guiada collections) leverage positional encoding -- users process ordinal position faster than reading a separate "current item" label.

4. **Norman, D. A. (2013).** *The Design of Everyday Things.* Revised ed. -> Duplicate detection warning as non-blocking inline banner follows the error prevention principle (prevent the error before it happens) while respecting user agency (the user can override). Constraints that prevent errors without removing user control.

5. **Nielsen, J. (1994).** "Usability inspection methods." *Conference Companion on Human Factors in Computing Systems.* -> Heuristic #5 (Error Prevention): the deduplication banner appears after the user finishes typing (debounce), not prematurely. Heuristic #9 (Help users recognize and recover): the banner includes "Ver este" to inspect the candidate and "Criar mesmo assim" to proceed.

6. **Cooper, A., Reimann, R. & Cronin, D. (2014).** *About Face.* 4th ed. -> Breadcrumb navigation within the collection modal for nested collections: users maintain orientation within a modal hierarchy without spawning nested modals (maximum stack depth = 1 modal). Content replacement within the same modal with breadcrumb trail preserves context.

7. **Galitz, W. O. (2007).** *The Essential Guide to User Interface Design.* 3rd ed. -> Single-column scrollable list for collection items (not grid): personal collections benefit from scannable titles with inline metadata, not visual thumbnails that compete for attention.

8. **Nielsen Norman Group (2023).** "Indicators, Validations, and Notifications: Pick the Correct Communication Option." -> Duplicate detection uses an inline banner (passive warning), not a modal dialog (which would block). Warning styling must differ from error styling -- amber/yellow vs. red.

9. **Miller, G. A. (1956).** "The magical number seven." *Psychological Review.* -> Breadcrumb segments limited to display the last 3 levels; overflow for deeper nesting uses ellipsis (...) to stay within working memory limits.

### 1. AcervoView Filter Tabs (modify existing)

**Current:** Single list of all content items.
**Phase 4:** Add segmented filter control at the top of the content area.

```
[Todos] [Itens] [Coletaneas]
```

**Implementation:** Three `ToggleButton` controls in a horizontal `StackPanel`, functioning as a radio group (only one active at a time).

| Property | Value |
|----------|-------|
| Position | Below header row, above content list (Grid.Row="1.5" -- new row) |
| Margin | 24px left, 0 top, 24px right, 8px bottom |
| Button padding | 12px horizontal, 6px vertical |
| Active state | `SystemAccentColor` at `Opacity="0.15"` background, `SystemAccentColor` foreground, `FontWeight="SemiBold"` |
| Inactive state | Transparent background, default foreground, `FontWeight="Regular"` |
| Font size | 13px |
| Spacing between buttons | 4px |
| Default selection | "Todos" |

**Transition behavior (Claude's Discretion area):** Content list crossfades with 150ms opacity transition when switching tabs. No horizontal slide -- reduces motion sickness risk (WCAG 2.3.3 Animation from Interactions).

**AutomationProperties:**
- Container: `AutomationProperties.Name="Filtro por tipo de conteudo"`
- Each button: `AutomationProperties.Name="Mostrar {tipo}"`

### 8. Deduplication Warning Banner (D-07, D-08, D-09)

**Trigger:** User types in the Titulo field of `CriarConteudoView`. After 500ms debounce and minimum 3 characters, the system checks for duplicates.

**Position:** Directly below the Titulo field, above the error message area.

**High confidence (URL match -- D-07):**

```
+--------------------------------------------+
| [!] Fonte identica detectada               |  <- 13px SemiBold, amber
|     "Duna" criado em 02/04/2026            |  <- 14px Body
|     [Ver este]  [Criar mesmo assim]         |  <- Links/buttons
+--------------------------------------------+
```

**Medium confidence (normalized title match -- D-07):**

```
+--------------------------------------------+
| [!] Titulo semelhante encontrado           |  <- 13px SemiBold, amber
|     "Duna - Frank Herbert" criado em 01/04  |  <- 14px Body
|     [Ver este]  [Criar mesmo assim]         |
+--------------------------------------------+
```

| Element | Spec |
|---------|------|
| Container | `Border`, `Background=#FEFCE8`, `BorderBrush=#CA8A04` at Opacity="0.3", `BorderThickness="1"`, `CornerRadius="6"`, padding 12px |
| Icon | Warning triangle unicode, `#CA8A04`, 16px |
| Heading | 13px SemiBold, `#CA8A04`. Text varies by confidence level (D-09) |
| Candidate info | 14px Body, default foreground. Shows title + creation date |
| "Ver este" button | Link-style button (underlined text, no border, accent color). Opens candidate's `ConteudoDetalheWindow` in overlay |
| "Criar mesmo assim" button | Link-style button (underlined text, no border, default foreground at Opacity="0.7"). Dismisses the banner and allows creation to proceed |
| Animation | Slide-down + fade-in, 200ms, when duplicate detected. Slide-up + fade-out when dismissed or title changes |
| Multiple candidates | If >1 candidate found, show only the highest-confidence match. Additional matches noted as "e {n} outro(s) semelhante(s)" |
| AutomationProperties | Container: `AutomationProperties.Name="Aviso de possivel duplicata"`. "Ver este": `AutomationProperties.Name="Ver conteudo existente: {titulo}"`. "Criar mesmo assim": `AutomationProperties.Name="Ignorar aviso e criar conteudo"` |

**Debounce and threshold:**
- Debounce: 500ms after last keystroke (longer than category autocomplete's 200ms -- duplicate check is heavier)
- Minimum characters: 3 (below this, too many false positives for title matching)
- Normalized comparison: trim + lowercase + remove diacritics + remove punctuation (D-07)

## Interaction Contracts

### Filter Tab Switching (D-01)

1. User clicks filter tab ("Todos", "Itens", "Coletaneas")
2. `AcervoViewModel` updates `PapelFiltro` property
3. `ListarConteudosQuery` re-executes with `PapelConteudo?` filter parameter
4. Content list refreshes with 150ms crossfade transition
5. Pagination resets to page 1
6. Status text updates: "Mostrando {n} de {total} {tipo}"

### Collection Card -> Modal Flow (D-02)

1. User clicks eye icon on collection card
2. `AcervoViewModel` creates `ColetaneaDetalheViewModel` via factory
3. `ColetaneaDetalheModal` opens as modal (`ShowDialog`)
4. Modal loads collection items via paginated query
5. Breadcrumb shows single segment (collection title, non-clickable)
6. For Guiada: "Proximo item" indicator appears if there are unconsumed items

### Nested Collection Navigation (D-02 breadcrumb)

1. User clicks a sub-collection item in the item list
2. Item list content replaces with the sub-collection's items (no new modal)
3. Breadcrumb adds new segment: "Raiz > Sub-collection"
4. User clicks ancestor breadcrumb segment to navigate back
5. Maximum display: 3 breadcrumb segments visible; overflow uses ellipsis with dropdown

### Add Item to Collection Flow (D-04)

1. User clicks "Adicionar item" in collection modal
2. Inline form expands below item list
3. User searches for content via `AutoCompleteBox` (paginated results, 8 max visible)
4. User selects item from results
5. If selected item is a collection: backend runs cycle detection via `ObterDescendentesAsync`
6. If cycle detected: inline error message appears (destructive color). Item NOT added
7. If no cycle: user clicks "Adicionar" -> item appended to collection
8. For Guiada: item gets next sequential position automatically
9. Item list refreshes. Inline form stays open for adding more items

### Contextual Annotation Flow (D-05, D-06)

1. User clicks annotation icon on item row in collection modal
2. Inline text area expands below the item row
3. User types annotation text
4. On blur (focus leaves text area): annotation auto-saves
5. Visual feedback: brief checkmark icon (1 second) confirms save
6. Clicking annotation icon again collapses the editor
7. Icon changes: filled pencil (has content) vs. outline pencil (empty)

### Source Reorder Flow (D-12)

1. User clicks up/down arrow on a source row in the Fontes section
2. Source moves one position in the indicated direction
3. Priority numbers re-assign: 1, 2, 3...
4. Brief accent highlight flash on moved row (200ms)
5. Up arrow disabled on first item; down arrow disabled on last item
6. Changes included in modal's dirty state -- saved with "Salvar"

### Source Addition Flow (D-12, D-13)

1. User clicks "Adicionar fonte" in Fontes section
2. Inline form expands: type dropdown + value text field
3. User selects source type from dropdown (default: Url)
4. Watermark on value field updates based on selected type
5. User enters value
6. User clicks "Adicionar" -> source added at end of list (lowest priority)
7. Priority auto-assigned as next number
8. Inline form stays open for adding more sources

### Cover Image Flow (D-15, D-16)

1. User clicks "Escolher imagem" in Identificacao section
2. OS file picker opens (filter: jpg, jpeg, png, webp)
3. User selects file
4. If file > 10MB: inline error "A imagem excede o limite de 10 MB"
5. If valid: image preview appears at 120x160px
6. "Remover" button becomes visible
7. Image change included in modal's dirty state
8. On "Salvar": image persisted via `Conteudo.AdicionarImagem()`

### Deduplication Check Flow (D-07, D-08, D-09)

1. User types in Titulo field of `CriarConteudoView`
2. After 500ms debounce and >= 3 characters: system runs dedup check
3. Check order: (a) URL match on any source (high confidence), (b) normalized title match (medium confidence)
4. If match found: warning banner slides in below Titulo field
5. Banner shows confidence level heading + candidate info
6. User clicks "Ver este": candidate's `ConteudoDetalheWindow` opens
7. User clicks "Criar mesmo assim": banner dismisses, creation proceeds normally
8. If title changes significantly (edit distance > 3 from matched title): banner auto-dismisses
9. If no match: no banner shown

### Collection Deletion Flow

1. User clicks "Excluir" in collection modal footer
2. Confirmation dialog: "'{Titulo}' sera excluida permanentemente. Os itens dentro da coletanea NAO serao excluidos -- apenas a organizacao sera perdida."
3. On confirm: `ExcluirColetaneaCommand` -> collection deleted, items remain in acervo
4. Modal closes, AcervoView refreshes
5. On cancel: dialog closes, no action

### Keyboard Navigation (extends Phase 3)

- Tab order in collection modal: breadcrumb -> item list -> add item button -> footer buttons
- Arrow keys in item list: navigate between items (up/down)
- Enter on item row: if sub-collection, navigate into it
- Enter on annotation icon: toggle annotation editor
- Escape in annotation editor: close editor (save on blur has already saved)
- Tab between source reorder buttons: up arrow, down arrow, remove (left to right)
