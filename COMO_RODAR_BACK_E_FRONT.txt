=====================================================================
COMO RODAR — Diário de Bordo
=====================================================================

Todos os comandos assumem que você está na raiz do repositório.


=====================================================================
1. VIA DOCKER COMPOSE (recomendado)
=====================================================================

PRÉ-REQUISITOS
  - Docker e Docker Compose
  - PostgreSQL instalado e acessível na máquina hospedeira
  - Banco "diariodebordo" criado (ver 1.1)
  - PostgreSQL aceitando conexões TCP da rede Docker (ver 1.2)

---------------------------------------------------------------------
1.1 — CRIAR BANCO (apenas na primeira vez)
---------------------------------------------------------------------
   PGPASSWORD=<senha-do-postgres> psql -h 127.0.0.1 -U postgres \
     -c "CREATE DATABASE diariodebordo;"

---------------------------------------------------------------------
1.2 — CONFIGURAR POSTGRESQL PARA DOCKER (apenas na primeira vez)
---------------------------------------------------------------------
1. Habilitar listen_addresses:
   sudo bash -c "echo \"listen_addresses = '*'\" >> /etc/postgresql/16/main/postgresql.conf"

2. Permitir rede Docker no pg_hba.conf:
   sudo bash -c "echo \"host    all    postgres    172.0.0.0/8    scram-sha-256\" >> /etc/postgresql/16/main/pg_hba.conf"
   (172.0.0.0/8 cobre as subnets Docker: 172.17.x, 172.18.x, etc.)

3. Reiniciar:
   sudo systemctl restart postgresql

---------------------------------------------------------------------
1.3 — APLICAR MIGRATIONS (primeira vez ou após novas migrations)
---------------------------------------------------------------------
   cd backend
   ConnectionStrings__DefaultConnection="Host=127.0.0.1;Port=5432;Database=diariodebordo;Username=postgres;Password=<senha>;SSL Mode=Disable" \
   Jwt__Key="diariodebordo-dev-key-min-32-characters-ok" \
   dotnet ef database update \
     --project src/DiarioDeBordo.Persistence \
     --startup-project src/DiarioDeBordo.Api
   cd ..

Nota: em Development, o seed (12 obras de exemplo + usuário admin)
é aplicado automaticamente na primeira inicialização da API.

---------------------------------------------------------------------
1.4 — SUBIR TUDO
---------------------------------------------------------------------
   docker compose -f docker/docker-compose.yml up --build

Parar: Ctrl+C ou  docker compose -f docker/docker-compose.yml down


=====================================================================
2. DESENVOLVIMENTO LOCAL (sem Docker)
=====================================================================

PRÉ-REQUISITOS
  - .NET SDK 10.x
  - Node.js 22.x + npm
  - PostgreSQL com banco "diariodebordo" criado (ver 1.1)

---------------------------------------------------------------------
2.1 — Backend
---------------------------------------------------------------------
   cd backend
   dotnet restore
   ConnectionStrings__DefaultConnection="Host=127.0.0.1;Port=5432;Database=diariodebordo;Username=postgres;Password=<senha>;SSL Mode=Disable" \
   Jwt__Key="diariodebordo-dev-key-min-32-characters-ok" \
   dotnet run --project src/DiarioDeBordo.Api

---------------------------------------------------------------------
2.2 — Frontend
---------------------------------------------------------------------
   cd frontend
   npm install
   npx ng serve

4. AUTENTICAÇÃO
=====================================================================
Usuário admin criado pelo seed de Development:
  login: admin
  senha: camaradinha@123

Opção A — Frontend:
  1. Abra http://localhost:4200
  2. Vá em "Configurações"
  3. Informe a URL da API: http://localhost:5000
  4. Preencha login/senha e clique "Entrar"
  5. Acesse "Obras"

Opção B — curl:
   curl -X POST http://localhost:5000/api/auth/login \
     -H "Content-Type: application/json" \
     -d '{"login":"admin","senha":"camaradinha@123"}'

  Copie o "token" e cole em:
  Configurações > Token manual > campo "Token JWT (Bearer)".

=====================================================================
OBSERVAÇÕES
=====================================================================
- A senha do PostgreSQL no docker-compose.yml está em texto claro
  apenas para ambiente de desenvolvimento local. Em outros ambientes,
  use secrets (Docker Secrets, variáveis de ambiente no CI/CD, vault).

- O token JWT obtido via login é armazenado apenas no localStorage do
  navegador (não é enviado a terceiros).

- O seed de dados (12 obras de exemplo + usuário admin) é aplicado
  automaticamente apenas em ASPNETCORE_ENVIRONMENT=Development e
  somente quando as tabelas estiverem vazias. Não afeta dados existentes.

- LGPD: o usuário admin inicia com Requer2FA=false para facilitar o
  desenvolvimento. Habilitar 2FA antes de expor em produção.
